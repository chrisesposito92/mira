    ```typescript
    async function performSave(): Promise<void> {
        if (!diagramStore.currentDiagram) return;

        // Capture version at time of save initiation
        const thisVersion = ++saveVersion;
        saveStatus = 'saving';

        // Read current content inside this function to avoid stale closure (pitfall 3)
        await diagramStore.updateContent(service, diagramStore.currentDiagram.content);

        // Staleness check: if another save was initiated after us, discard our result
        if (thisVersion !== saveVersion) return;

        if (!diagramStore.error) {
            saveStatus = 'saved';
            // Check if thumbnail is due (every 10 seconds, not every save)
            const now = Date.now();
            if (now - lastThumbnailTime > 10_000) {
                lastThumbnailTime = now;
                await generateAndPersistThumbnail();
            }
        } else {
            saveStatus = 'error';
            // Toast for errors only (D-15)
            toast.error('Changes could not be saved. Check your connection and try again.');
        }
    }
    ```

    **Thumbnail generation function (D-16) -- throttled:**
    Review concern addressed (both reviewers MEDIUM): Thumbnail generation is now decoupled from every 500ms save. It fires:
    1. Every 10 seconds during active editing (checked in performSave)
    2. On navigation away (beforeNavigate)
    3. Best-effort -- failures don't affect saves

    ```typescript
    async function generateAndPersistThumbnail(): Promise<void> {
        if (!diagramStore.currentDiagram) return;

        try {
            const svgElement = document.querySelector('svg[role="img"]');
            if (!svgElement) return;

            const svgData = new XMLSerializer().serializeToString(svgElement);
            const blob = new Blob([svgData], { type: 'image/svg+xml;charset=utf-8' });
            const url = URL.createObjectURL(blob);

            const thumbnail = await new Promise<string | null>((resolve) => {
                const img = new Image();
                img.onload = () => {
                    const canvas = document.createElement('canvas');
                    canvas.width = 300;
                    canvas.height = 200;
                    const ctx = canvas.getContext('2d')!;
                    ctx.drawImage(img, 0, 0, 300, 200);
                    URL.revokeObjectURL(url);
                    resolve(canvas.toDataURL('image/png', 0.8));
                };
                img.onerror = () => {
                    URL.revokeObjectURL(url);
                    resolve(null);
                };
                img.src = url;
            });

            if (thumbnail) {
                // Persist thumbnail via a separate update call
                await service.update(diagramStore.currentDiagram.id, { thumbnail_base64: thumbnail });
            }
        } catch {
            // Thumbnail generation is best-effort -- don't fail the save
            console.warn('Thumbnail generation failed');
        }
    }
    ```

    **Important implementation notes:**
    - `saveVersion` is a monotonically increasing counter. Each call to `performSave` captures its version. If another save starts before the first completes, the first save's version check fails and it silently discards its result. This prevents stale save responses from overwriting newer edits (addresses HIGH review concern).
    - The `dirty` state is set immediately when content changes, before the debounce fires. This means the save indicator shows "unsaved changes" during the 500ms window, not a misleading "Saved" (addresses Codex LOW concern).
    - `beforeNavigate` flushes any pending debounced save. Since `performSave` is async, the flush is best-effort for `willUnload` events (browser tab close), but covers SvelteKit route transitions fully.
    - Thumbnail generation every 10 seconds is controlled by `lastThumbnailTime` timestamp comparison. This dramatically reduces CPU and DB write volume compared to every 500ms (addresses both reviewers' MEDIUM concern).
    - Canvas size: 300x200 pixels, PNG quality 0.8 (per RESEARCH.md pattern).

    ```typescript
    // Test the save versioning logic in isolation
    describe('Auto-save version counter logic', () => {
        it('increments version on each save call', () => {
            let saveVersion = 0;
            const v1 = ++saveVersion;
            const v2 = ++saveVersion;

            // Simulate another save starting

            // Original save completes -- but version has advanced
            const isStale = capturedVersion !== saveVersion;

        });

        it('does not detect stale when no concurrent save', () => {
            let saveVersion = 0;

            expect(isStale).toBe(false);
    ```
