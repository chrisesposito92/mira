# Phase 3 -- UI Design Contract

> Visual and interaction contract for the Configurator UI phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-svelte 1.1 (bits-ui 2.16 headless primitives) |
| Preset | default style, zinc baseColor (`frontend/components.json`) |
| Component library | bits-ui (underlying shadcn-svelte) |
| Icon library | lucide-svelte 0.575 |
| Font | Inter (system font stack: `'Inter', -apple-system, BlinkMacSystemFont, sans-serif`) |
| CSS framework | Tailwind CSS v4.2 via `@theme inline` blocks (no `tailwind.config.js`) |
| Color variables | HSL CSS custom properties (e.g., `--primary: 240 5.9% 10%`) |
| Dark mode | mode-watcher (automatic, no manual class toggling) |

### Editor Page Layout (D-01, D-03)

```
+---------------------------------------------+
| Header: [<-] Customer Name  [Save Status]   |
+----------+----------------------------------+
|          |                                   |
| Sidebar  |   DiagramRenderer (live preview)  |
| 360px    |   (fills remaining width)         |
| fixed    |                                   |
|          |                                   |
| [Tabs]   |   Aspect ratio: 3:2 (1200x800)   |
| Systems  |   Contained in rounded border     |
| Connect. |   overflow-hidden                  |

|          |                                   |

```

- Sidebar: 360px fixed width, full viewport height minus header, `overflow-y: auto` via ScrollArea

- Header: Spans full width above the two-panel layout, contains back button, customer name heading, save status indicator
- Divider: `border-r` between sidebar and preview (uses `--border` color)

## Interaction Contract

### System Picker Interactions

| Action | Trigger | Visual feedback | State change |
|--------|---------|----------------|--------------|
| Search systems | Type in search input | Category list filters in real-time | Client-side filter on `name` field |
| Expand category | Click category header | Collapsible opens with slide animation | Category `isOpen` toggles |
| Add system | Click enabled SystemPickerItem | Item becomes dimmed + checkmark | `diagramStore.addSystem()` called |
| Click already-added system | Click dimmed item | No response (cursor: default) | No state change |
| Remove system | Click X button on added system item | System un-dims in picker, removed from diagram | `diagramStore.removeSystem()` called |
| Add custom system | Click "Add Custom System" button | AddCustomSystemDialog opens | Dialog handles submission |

### Connection Form Interactions

| Action | Trigger | Visual feedback | State change |
|--------|---------|----------------|--------------|
| Open form | Click "Add Connection" button | Inline form expands below button | `showConnectionForm = true` |
| Select source | Pick from Select dropdown | Dropdown closes, source shown | Form state updates |
| Select target | Pick from Select dropdown | Target dropdown excludes source | Form state updates |
| Toggle direction | Click Switch component | Switch animates, label changes "One-way" / "Two-way" | `direction` toggles |
| Select type | Click ToggleGroup item | Active item highlighted | `connection_type` updates |
| Auto-suggest type | Source or target is m3ter + other has `is_native_connector` | Type auto-selects "Native Connector" | `connection_type = 'native_connector'` (D-11) |
| Type label | Type in label Input | Suggestion dropdown appears below (D-12) | `label` updates |
| Select suggestion | Click suggestion item | Input fills with selected label | `label` set to suggestion |
| Submit connection | Click "Add" button | Form collapses, connection appears in list | `diagramStore.addConnection()` |
| Cancel form | Click "Cancel" button | Form collapses | Form state resets |
| Edit connection | Click edit icon on ConnectionListItem | Inline form re-opens pre-filled | Connection in edit mode |
| Delete connection | Click delete icon on ConnectionListItem | Connection removed from list | `diagramStore.removeConnection()` |

### Auto-Save Interactions (D-14, D-15)

| Action | Trigger | Visual feedback | State change |
|--------|---------|----------------|--------------|
| Content change detected | Any diagram content mutation | SaveStatusIndicator shows "Saving..." with spinner | Debounce timer starts (500ms) |
| Save in progress | Debounce fires | Spinner continues | `updateContent()` API call |
| Save success | API responds 200 | Checkmark icon + "Saved" text (fades after 2s to idle) | `saving = false`, thumbnail generated |
| Save error | API responds error | Red warning icon + "Save failed" text | Toast error shown (toasts only for errors per D-15) |
| Initial page load | Editor mounts | No save triggered | `$effect` skips first run |

### Thumbnail Generation (D-16)

| Step | Implementation |
|------|---------------|
| 1. Trigger | After successful auto-save completes |
| 2. Serialize SVG | `svgElement.outerHTML` from the DiagramRenderer DOM ref |
| 3. Create canvas | `document.createElement('canvas')` at 300x200 resolution |
| 4. Draw SVG | Create `Image` from SVG blob URL, draw to canvas on `onload` |
| 5. Export | `canvas.toDataURL('image/png', 0.8)` -> `thumbnail_base64` |
| 6. Persist | Second `service.update()` call with `{ thumbnail_base64 }` |

### Connection Type Auto-Suggestion Logic (D-11, CONN-06)

```
IF (source is m3ter hub OR target is m3ter hub)
  AND (the other system has is_native_connector === true in componentLibrary)
THEN set connection_type = 'native_connector' (user can override)
```

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Tab navigation | Sidebar tabs use shadcn Tabs component (built-in arrow key navigation) |
| Collapsible categories | Collapsible uses `aria-expanded`, `aria-controls` (built into bits-ui) |
| Search input | `role="search"` or `<search>` element wrapper, `aria-label="Search systems"` |
| System picker items | `role="button"`, `aria-disabled="true"` for already-added items |
| Connection form selects | shadcn Select provides built-in ARIA attributes |
| Direction toggle | shadcn Switch provides built-in `role="switch"`, `aria-checked` |
| Type selector | shadcn ToggleGroup provides built-in `role="group"` with `aria-pressed` |
| Save status | `aria-live="polite"` on the SaveStatusIndicator container |
| DiagramRenderer | Existing `role="img"` with `aria-label` (unchanged) |
| Delete/remove buttons | `aria-label="Remove {system name}"` / `aria-label="Delete connection to {target}"` |
| Thumbnail placeholder | `alt=""` on thumbnail img (decorative), `aria-hidden="true"` on placeholder icon |
