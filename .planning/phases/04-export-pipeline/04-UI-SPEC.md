### Header Bar Modification (D-01)

The only layout change is adding the ExportDropdown to the existing DiagramBuilder header bar. The header bar layout becomes:

```
+------------------------------------------------------------------+
| [<-]  Customer Name          [Save Status]  [Export v]            |
|       (subtitle if present)                                       |
+------------------------------------------------------------------+
```

- Left section (unchanged): Back button + customer name heading + optional subtitle
- Right section (modified): SaveStatusIndicator + ExportDropdown (8px gap between them)
- ExportDropdown aligns vertically centered with the back button and save indicator

### ExportDropdown Internal Layout

```
+-------------------+
| [Download] Export  |  <-- Button trigger (outline variant, size sm)
+-------------------+
        |
        v
+-------------------+
| [Image]  PNG      |  <-- DropdownMenu.Item
| [FileCode] SVG    |  <-- DropdownMenu.Item
+-------------------+
```

- Trigger: `Button` variant `outline`, size `sm`, with `Download` lucide icon (16px) + "Export" text

When diagram has zero system nodes:
- Export button renders with `disabled` attribute
- Button wrapped in `Tooltip.Root` / `Tooltip.Trigger` / `Tooltip.Content`
- Tooltip appears on hover/focus of the disabled button
- Dropdown does not open

## Interaction Contract

### Export Flow Interactions

| Action | Trigger | Visual feedback | State change |
|--------|---------|----------------|--------------|
| Open export dropdown | Click "Export" button | Dropdown menu opens below-right | `open = true` |
| Export as PNG | Click "PNG" dropdown item | Dropdown closes, browser save dialog appears | `exportDiagram('png')` called |
| Export as SVG | Click "SVG" dropdown item | Dropdown closes, browser save dialog appears | `exportDiagram('svg')` called |
| Export with empty diagram | Hover disabled button | Tooltip shows "Add at least one system to export" | No state change |
| Export failure | Canvas security error or font load failure | Toast error appears | `toast.error()` called |
| Keyboard: open dropdown | `Enter` or `Space` on focused button | Same as click | Same as click |
| Keyboard: select item | `Enter` or `Space` on focused item | Same as click | Same as click |
| Keyboard: close dropdown | `Escape` | Dropdown closes, focus returns to trigger | `open = false` |

### Export Pipeline Steps (D-02, D-07, D-08, D-09)

No visible progress UI. Export completes sub-second. Steps are internal:

#### PNG Export (EXPO-01)

| Step | Implementation |
|------|---------------|
| 1. Query SVG | `document.querySelector('svg[role="img"]')` |
| 2. Serialize | `new XMLSerializer().serializeToString(svgElement)` |
| 3. Inject font | Insert `<style>` block with base64 @font-face into serialized SVG string |
| 4. Create blob | `new Blob([svgString], { type: 'image/svg+xml;charset=utf-8' })` |
| 5. Load image | Create `Image`, set `src` to blob object URL, wait for `onload` |
| 6. Draw canvas | Create `<canvas>` at 2400x1600 (2x of 1200x800), `ctx.drawImage()` |
| 7. Export PNG | `canvas.toBlob(callback, 'image/png')` |
| 8. Download | Create `<a>` element, set `href` to blob URL, set `download` to filename, click, revoke URL |

#### SVG Export (EXPO-02)

| Step | Implementation |
|------|---------------|
| 1. Query SVG | `document.querySelector('svg[role="img"]')` |
| 2. Serialize | `new XMLSerializer().serializeToString(svgElement)` |
| 3. Inject font | Insert `<style>` block with base64 @font-face into serialized SVG string |
| 4. Add XML declaration | Prepend `<?xml version="1.0" encoding="UTF-8"?>` |
| 5. Download | Create blob (`image/svg+xml`), create `<a>` with blob URL, set `download` to filename, click, revoke URL |

### Font Injection Detail (EXPO-04, D-04, D-05)

The Inter Regular (400) woff2 file is bundled as a static asset in the frontend. At export time:

1. Import the font file as a static asset URL (Vite handles this via `import interWoff2 from '$lib/assets/inter-regular.woff2?url'`)
2. Fetch the font file, convert to base64 via `FileReader` or `btoa(String.fromCharCode(...))`
3. Construct the `@font-face` CSS rule:
   ```css
   @font-face {
     font-family: 'Inter';
     font-weight: 400;
     font-style: normal;
     src: url(data:font/woff2;base64,{BASE64_DATA}) format('woff2');
   }
   ```
4. Inject as the first child `<style>` element inside the serialized SVG's `<svg>` opening tag

Font loading strategy: **Lazy-load at first export** (not page load). Cache the base64 result in a module-level variable so subsequent exports in the same session skip the fetch.

### Filename Generation (D-06)

| Input | Output |
|-------|--------|
| `customer_name = "Acme Corp"` | `acme-corp.png` / `acme-corp.svg` |
| `customer_name = "AT&T / Wireless"` | `att-wireless.png` |
| `customer_name = ""`, `title = "Q4 Architecture"` | `q4-architecture.png` |
| `customer_name = ""`, `title = ""` | `integration-diagram.png` |

Slugify rules:
1. Lowercase the input
2. Replace `&` with empty string
3. Replace non-alphanumeric characters with hyphens
4. Collapse consecutive hyphens into one
5. Trim leading/trailing hyphens
6. Append file extension (`.png` or `.svg`)

- No confirmation dialogs. Export triggers an immediate browser download (D-02).

## Error Handling Contract

| Error Case | Detection | User Feedback | Recovery |
|------------|-----------|---------------|----------|
| SVG element not found | `querySelector` returns null | Toast: "Export failed. Try refreshing the page and exporting again." | User refreshes page |
| Canvas security error | `canvas.toBlob()` throws | Toast: "Export failed. Try refreshing the page and exporting again." | User refreshes page |
| Font file fetch failure | `fetch()` rejects or returns non-OK | Continue export without font injection. Toast: "Export completed without custom font. The diagram may use a fallback font." | Graceful degradation -- export still works |
| Image load failure | `img.onerror` fires during PNG canvas pipeline | Toast: "Export failed. Try refreshing the page and exporting again." | User refreshes page |
| Blob URL creation failure | `URL.createObjectURL()` throws | Toast: "Export failed. Try refreshing the page and exporting again." | User refreshes page |

Font injection failure is non-blocking. All other export failures show an error toast and abort the export. No partial files are downloaded.

## Accessibility Contract

| Requirement | Implementation |
|-------------|---------------|
| Export button | Standard shadcn Button -- keyboard accessible, focus ring |
| Dropdown menu | shadcn DropdownMenu provides built-in ARIA: `role="menu"`, `aria-expanded`, arrow key navigation, `Escape` to close |
| Dropdown items | `role="menuitem"` with keyboard Enter/Space activation (built into bits-ui) |
| Disabled state | `aria-disabled="true"` on button, tooltip accessible via focus (not just hover) |
| Tooltip | shadcn Tooltip provides `role="tooltip"`, `aria-describedby` linking (built into bits-ui) |
| Download trigger | Programmatic `<a>` click -- no accessibility impact (file save dialog is browser-native) |
| Icons | All lucide icons in dropdown items are decorative: `aria-hidden="true"` (lucide-svelte default) |
