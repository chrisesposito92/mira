    **`slugify(text: string): string`** (per D-06):
    - Lowercase the input
    - Replace `&` with empty string
    - Replace all non-alphanumeric characters with hyphens (`/[^a-z0-9]+/g` -> `'-'`)
    - Collapse consecutive hyphens (`/-+/g` -> `'-'`)
    - Trim leading/trailing hyphens (`/^-|-$/g` -> `''`)

    **`generateFilename(customerName: string, title: string, format: ExportFormat): string`** (per D-06):
    - Use `customerName.trim()` if non-empty, else `title.trim()` if non-empty, else `'integration-diagram'`
    - Return `${slugify(base)}.${format}`

    **`arrayBufferToBase64(buffer: ArrayBuffer): string`**:
    - Use a for-loop (NOT spread operator -- stack overflow on 160KB+ files per research Pitfall 3):
      ```typescript
      const bytes = new Uint8Array(buffer);
      let binary = '';
      for (let i = 0; i < bytes.byteLength; i++) {
        binary += String.fromCharCode(bytes[i]);
      }
      return btoa(binary);
      ```

    **Module-level font cache:**
    ```typescript
    let cachedFontBase64: string | null = null;

    async function loadFontBase64(): Promise<string> {
      if (cachedFontBase64) return cachedFontBase64;
      const fontUrl = new URL('../assets/fonts/inter-latin-wght-normal.woff2', import.meta.url);
      const response = await fetch(fontUrl);
      if (!response.ok) throw new Error('Font fetch failed');
      const buffer = await response.arrayBuffer();
      cachedFontBase64 = arrayBufferToBase64(buffer);
      return cachedFontBase64;
    }
    ```

    **`injectFont(svgString: string, fontBase64: string): string`**:
    - Build the `@font-face` style block:
      ```
      <style>
        @font-face {
          font-family: 'Inter';
          font-style: normal;
          font-display: swap;
          font-weight: 100 900;
          src: url(data:font/woff2;base64,${fontBase64}) format('woff2');
        }
      </style>
      ```
    - Insert after the opening `<svg ...>` tag using regex: `svgString.replace(/(<svg[^>]*>)/, '$1' + fontFaceBlock)`
    - Return the modified SVG string

    **`fixContextStroke(svgString: string): string`** (per research Pitfall 2 and Open Question 1):
    The SVG markers use `fill="context-stroke"` which may not render correctly when SVG is loaded as an Image for canvas rendering. Fix approach:

    1. Parse the SVG string to find all `<line` elements (connections) and extract their `stroke` attribute values. These are hex colors from CONNECTION_COLORS: `#00C853`, `#2196F3`, `#FF9800`, `#90A4AE`.
    2. For each unique stroke color found, create duplicate marker defs with explicit fill colors. For example, for color `#00C853`:
       - `<marker id="arrowhead-00C853" ...><path ... fill="#00C853" /></marker>`
       - `<marker id="source-dot-00C853" ...><circle ... fill="#00C853" /></marker>`
    3. Inject these new marker defs into the `<defs>` block (insert before `</defs>`).
    4. Rewrite each `<line` element's `marker-start` and `marker-end` attributes to reference the color-specific markers:
       - A line with `stroke="#00C853"` and `marker-end="url(#arrowhead)"` becomes `marker-end="url(#arrowhead-00C853)"`
       - Same for `marker-start="url(#source-dot)"` becomes `marker-start="url(#source-dot-00C853)"`

    Use regex-based string manipulation (not DOM parsing) since this operates on the serialized SVG string. The original `context-stroke` markers in `<defs>` can remain (they're harmless; the lines no longer reference them).

    Steps:
    - Extract unique stroke colors: `const strokeMatches = svgString.matchAll(/<line[^>]*stroke="(#[A-Fa-f0-9]{6})"[^>]*/g)`
    - For each unique color, build arrowhead and source-dot marker defs with that color as explicit fill
    - Inject before `</defs>`: `svgString.replace('</defs>', newMarkers + '</defs>')`
    - For each line element, extract its stroke color and rewrite marker references: replace `url(#arrowhead)` with `url(#arrowhead-{colorHex})` and `url(#source-dot)` with `url(#source-dot-{colorHex})` where colorHex is the stroke color with `#` stripped

    **`ensureSvgDimensions(svgString: string): string`** (per research Pitfall 5):
    - Check if the `<svg` tag already has `width` and `height` attributes
    - If not, add `width="1200" height="800"` (from CANVAS_WIDTH/CANVAS_HEIGHT constants)
    - Use regex: if `/<svg[^>]*width=/.test(svgString)` is false, then `svgString.replace(/<svg/, '<svg width="1200" height="800"')`

    **`downloadSvg(svgString: string, filename: string): void`**:
    - Prepend XML declaration: `'<?xml version="1.0" encoding="UTF-8"?>\n' + svgString`
    - Create Blob with type `'image/svg+xml;charset=utf-8'`
    - Call `triggerDownload(blob, filename)`

    **`async downloadPng(svgString: string, filename: string): Promise<void>`** (per D-07, D-08):
    - Use data URL (NOT createObjectURL) to avoid canvas taint (per research Pitfall 1):
      `const base64Svg = btoa(unescape(encodeURIComponent(svgString)));` (handles Unicode per Pitfall 6)
      `const dataUrl = 'data:image/svg+xml;base64,' + base64Svg;`
    - Create `new Image()`, set `img.width = 1200`, `img.height = 800`
    - Wait for `img.onload` via Promise; reject on `img.onerror`
    - Create canvas: `canvas.width = 2400; canvas.height = 1600;` (2x resolution per D-07)
    - Draw: `ctx.drawImage(img, 0, 0, 2400, 1600)`
    - Get blob: `canvas.toBlob(resolve, 'image/png')`
    - Call `triggerDownload(blob, filename)`

    **`triggerDownload(blob: Blob, filename: string): void`**:
    - `const url = URL.createObjectURL(blob)`
    - Create anchor: `const a = document.createElement('a'); a.href = url; a.download = filename;`
    - Append to body, click, remove from body, revoke URL

    **`async exportDiagram(format: ExportFormat, options: ExportOptions): Promise<void>`** (main entry point):
    - Query SVG: `document.querySelector('svg[role="img"]')` -- throw Error if null
    - Serialize: `new XMLSerializer().serializeToString(svgElement)`
    - Ensure dimensions: `ensureSvgDimensions(serialized)`
    - Load font (with try/catch -- font failure is non-blocking per error handling contract):
      ```typescript
      let withFont = withDimensions;
      try {
        const fontBase64 = await loadFontBase64();
        withFont = injectFont(withDimensions, fontBase64);
      } catch {
        // Font injection failed -- continue without font.
        // Caller handles the toast notification.
        console.warn('Font injection failed, exporting without embedded font');
      }
      ```
    - Fix context-stroke: `const fixed = fixContextStroke(withFont)`
    - Branch on format:
      - `'svg'`: call `downloadSvg(fixed, filename)`
      - `'png'`: call `await downloadPng(fixed, filename)`
