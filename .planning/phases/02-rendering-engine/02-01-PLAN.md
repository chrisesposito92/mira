    RED phase -- write tests first in diagram-layout.test.ts:

    - Test 1: `layoutDiagram` with empty systems returns hub at (600, 400) and prospect at (600, 60)
    - Test 2: `layoutDiagram` with a single system in category "CRM" places it in the first zone (left, angle=PI), not overlapping hub or prospect
    - Test 3: Systems with same category are grouped into a PositionedGroup (groups array has entry)
    - Test 4: Systems with null category are placed as standalone (standalone array has entry)
    - Test 5: Multiple categories distribute across zones in order of display_order from componentLibrary
    - Test 6: Prospect system (the one with `component_library_id === null` and `category === null` and name matches "Your Product/Platform" pattern, or first system with `id === 'prospect'`) is positioned at top (y near PROSPECT_Y)
    - Test 7: All positioned nodes have x within [0, CANVAS_WIDTH] and y within [0, CANVAS_HEIGHT]
    - Test 8: Function is deterministic -- same input produces identical output
    - Test 9: With many systems (15+), card sizes scale down proportionally (reflow per D-02)
    - Test 10: `parseMonogram` returns {initials, color} for valid "monogram:AB:#FF5722" format
    - Test 11: `parseMonogram` returns null for non-monogram strings (base64 data, null, empty)
    - Test 12: `getConnectionMidpoint` returns midpoint of two coordinates
    - Test 13: `estimatePillWidth` returns positive number proportional to text length

    GREEN phase -- implement in diagram-layout.ts:

    Zone-based layout algorithm per D-01/D-02/D-03:
    1. Identify prospect system (category null, no component_library_id, or explicitly named). Position at (HUB_CENTER_X, PROSPECT_Y).
    2. Group remaining systems by category using componentLibrary display_order.
    3. Assign categories to 5 zones: left (PI), bottom-left (PI*0.7), bottom (PI*0.5), bottom-right (PI*0.3), right (0). Extra categories wrap cyclically.
    4. For each zone, compute center position using cos/sin from hub center with radial distance ~250.
    5. Categories with 2+ systems become PositionedGroup. Single-system categories and null-category systems become standalone.
    6. Compute card widths/heights, scaling proportionally when total system count exceeds 15.
    7. Ensure all nodes fit within CANVAS_PADDING from edges.

    Also implement helper functions:
    - `parseMonogram(logo_base64: string | null): { initials: string; color: string } | null`
    - `getConnectionMidpoint(x1, y1, x2, y2): { x: number; y: number }`
    - `estimatePillWidth(text: string, fontSize: number): number`
  </behavior>
  <implementation>
    The layout function signature:
    ```typescript
    export function layoutDiagram(
      content: DiagramContent,
      componentLibrary: ComponentLibraryItem[]
    ): LayoutResult
    ```

    Key implementation details:
    - Pure function, no side effects, no DOM access
    - Uses Math.cos/Math.sin for zone positions
    - Radial distance from hub: 250 (base), adjusted by zone count
    - Group card sizing: width = SYSTEM_CARD_WIDTH * cols + padding, height = rows * (LOGO_SIZE + gap) + header
    - Scale factor: when systems > 15, multiply card dimensions by (15 / systemCount)^0.5
    - Monogram regex: `/^monogram:([A-Z]{1,3}):(#[0-9A-Fa-f]{6})$/`
  </implementation>
