    GREEN phase -- implement in diagram-layout.ts:

    Zone-based layout algorithm per D-01/D-02/D-03:
    1. Identify prospect system by `role === 'prospect'` first. Fallback: `component_library_id === null && category === null`. Position at (HUB_CENTER_X, PROSPECT_Y).
    2. Group remaining systems by category using componentLibrary display_order.
    3. Assign categories to 5 zones: left (PI), bottom-left (PI*0.7), bottom (PI*0.5), bottom-right (PI*0.3), right (0). Extra categories wrap cyclically.
    4. For each zone, compute center position using cos/sin from hub center with radial distance ~250.
    5. Categories with 2+ systems become PositionedGroup. Single-system categories and null-category systems become standalone.
    6. Compute card widths/heights, scaling proportionally when total system count exceeds 15.
    7. Ensure all nodes fit within CANVAS_PADDING from edges.
    8. BBox buffering: when a group has >4 systems, increase its radial distance by `(systemCount - 4) * 15` to prevent overlap with adjacent zones.
    9. Build `nodePositions` map with bounding box for every node (including hub keyed as 'hub').

    Also implement helper functions:
    - `parseMonogram(logo_base64: string | null): { initials: string; color: string } | null`
    - `getConnectionMidpoint(x1, y1, x2, y2): { x: number; y: number }`
    - `estimatePillWidth(text: string, fontSize: number): number`
    - `truncateSvgText(text: string, maxChars: number): string` -- returns text unchanged if <= maxChars, otherwise truncates and appends "..."
    - `computeEdgeAnchor(rect: {x, y, width, height}, targetX, targetY): {x: number, y: number}` -- given a node bounding box and the coordinates of a target point, returns the point on the rect edge where a line from rect center to target intersects. Uses parametric line-rect intersection.

    Key implementation details:
    - Pure function, no side effects, no DOM access
    - Uses Math.cos/Math.sin for zone positions
    - Radial distance from hub: 250 (base), adjusted by zone count and BBox size
    - Group card sizing: width = SYSTEM_CARD_WIDTH * cols + padding, height = rows * (LOGO_SIZE + gap) + header

    - Monogram regex: `/^monogram:([A-Z]{1,3}):(#[0-9A-Fa-f]{6})$/`
    - `computeEdgeAnchor`: parametric line from center of rect to target; find t where line crosses rect boundary
    - `nodePositions` map: after computing all positions, iterate through hub, prospect, groups (each system), and standalone to build flat lookup
