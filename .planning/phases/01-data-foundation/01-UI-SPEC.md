# Phase 1 — UI Design Contract

> Visual and interaction contract for Phase 1's minimal frontend surfaces. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

**Phase 1 frontend scope is narrow:** A sidebar nav item, a diagram list page with create/delete, and a create-diagram dialog. All patterns clone existing MIRA conventions (Dashboard page, ProjectCard, CreateProjectDialog, EmptyState). No novel visual patterns are introduced.

## Color

All colors use the existing shadcn-svelte zinc theme HSL variables declared in `app.css`. Phase 1 introduces zero custom colors.

| Role | CSS Variable | Light Value | Dark Value | Usage in Phase 1 |
|------|-------------|-------------|------------|-------------------|
| Dominant (60%) | `--background` | `hsl(0 0% 100%)` | `hsl(240 10% 3.9%)` | Page background |
| Secondary (30%) | `--card` / `--sidebar` | `hsl(0 0% 100%)` / `hsl(0 0% 98%)` | `hsl(240 10% 3.9%)` / `hsl(240 5.9% 10%)` | Diagram cards, sidebar |
| Accent (10%) | `--primary` | `hsl(240 5.9% 10%)` | `hsl(0 0% 98%)` | See reserved-for list below |
| Destructive | `--destructive` | `hsl(0 72.2% 50.6%)` | `hsl(0 62.8% 30.6%)` | Delete diagram button only |
| Muted | `--muted-foreground` | `hsl(240 3.8% 46.1%)` | `hsl(240 5% 64.9%)` | Card description text, metadata, empty state body |
| Border | `--border` | `hsl(240 5.9% 90%)` | `hsl(240 3.7% 15.9%)` | Card borders, EmptyState dashed border |

**Accent reserved for (10% rule):**
1. "New Diagram" primary CTA button (`bg-primary`)
2. "Create Diagram" submit button in dialog (`bg-primary`)
3. Active sidebar nav item highlight (`sidebar-primary`)
4. Card hover border (`hover:border-primary/50` -- matches ProjectCard pattern)

## Interaction Contract

### Sidebar Navigation

| Interaction | Behavior |
|-------------|----------|
| Click "Diagrams" nav item | Navigate to `/diagrams/` |
| Active state | `isActive()` returns true when path starts with `/diagrams` (matches existing `isActive` pattern in AppSidebar) |

### Diagram List

| Interaction | Behavior |
|-------------|----------|
| Page load | Fetch diagrams from API, show skeleton loading state (3 skeleton cards in grid), then render DiagramCards |
| Click "New Diagram" button | Open CreateDiagramDialog |
| Click DiagramCard | No navigation in Phase 1 (no builder route yet). Card is non-interactive. Clicking does nothing. |
| Click delete icon on card | Open DeleteDiagramDialog with diagram name |

### Create Dialog

| Interaction | Behavior |
|-------------|----------|
| Submit with valid data | POST to API, close dialog, prepend new diagram to list, show success toast ("Diagram created") |
| Submit with empty customer_name | Prevent submission (HTML required attribute + disabled button check) |
| Submit failure | Show error toast (sonner), keep dialog open to preserve user input |
| Cancel or close | Reset form fields, close dialog |
| While submitting | Button disabled, text shows "Creating..." |

### Delete Confirmation

| Interaction | Behavior |
|-------------|----------|
| Confirm delete | DELETE to API, remove card from list with no animation, show success toast ("Diagram deleted") |
| Cancel | Close dialog, no action |
| Delete failure | Show error toast, close dialog |

### Loading States

| State | Visual |
|-------|--------|
| Initial page load | 3 Skeleton cards in the grid layout (Skeleton components matching card dimensions) |
| Create in progress | Button disabled + "Creating..." text |
| Delete in progress | AlertDialog Action button disabled + "Deleting..." text |

## State Management Contract

### DiagramStore (`$lib/stores/diagrams.svelte.ts`)

Follows existing class-based singleton pattern (matches `projectStore`, `authStore`).

| Field | Type | Reactivity | Purpose |
|-------|------|------------|---------|
| `diagrams` | `Diagram[]` | `$state` | Raw diagram list from API |
| `loading` | `boolean` | `$state` | Loading indicator |
| `error` | `string \| null` | `$state` | Last error message |
| `sortedDiagrams` | `Diagram[]` | `$derived` | Sorted by `updated_at` descending |

| Method | Signature | Behavior |
|--------|-----------|----------|
| `createDiagram` | `(service, data) => Promise<Diagram \| null>` | POST, update list, return created or null |
| `deleteDiagram` | `(service, id) => Promise<boolean>` | DELETE, remove from list, return success |

### DiagramService (`$lib/services/diagrams.ts`)

Factory function pattern: `createDiagramService(client: ApiClient)`.

| Method | Endpoint | Purpose |
|--------|----------|---------|
| `listDiagrams()` | `GET /api/diagrams` | Fetch user's diagrams |
| `createDiagram(data)` | `POST /api/diagrams` | Create diagram |
| `deleteDiagram(id)` | `DELETE /api/diagrams/{id}` | Delete diagram |

## Accessibility Contract

| Element | Requirement |
|---------|-------------|
| Sidebar nav item | Standard `<a>` tag inside SidebarMenuButton -- keyboard navigable by default |
| DiagramCard delete button | `aria-label="Delete diagram {name}"` on the icon button |
| CreateDiagramDialog | Focus trapped by bits-ui Dialog primitive; first input receives focus on open |
| DeleteDiagramDialog | Focus trapped by bits-ui AlertDialog primitive; cancel button receives initial focus |
| Loading skeletons | `aria-hidden="true"` with `role="status"` wrapper and screen-reader-only "Loading diagrams" text |
| Empty state | Semantic heading (`h3`) + descriptive paragraph; CTA button is keyboard-focusable |
