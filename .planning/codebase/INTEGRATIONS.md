# External Integrations

**Analysis Date:** 2026-03-22

## APIs & External Services

**m3ter Billing Platform:**
- Config API (`api.m3ter.com`) — create/manage Products, Meters, Aggregations, PlanTemplates, Plans, Pricing, Accounts, AccountPlans
  - SDK/Client: custom `M3terClient` in `backend/app/m3ter/client.py` + official `m3ter>=0.8.0` SDK
  - Auth: OAuth2 client credentials grant (HTTP Basic auth to `/oauth/token`); tokens cached 4h50m (class-level cache on `M3terClient`)
  - Credentials stored encrypted in DB; decrypted at push time via Fernet (`backend/app/m3ter/encryption.py`)
  - Env vars: `M3TER_CLIENT_ID`, `M3TER_CLIENT_SECRET`, `M3TER_ORG_ID` — stored per org in `org_connections` table
- Ingest API (`ingest.m3ter.com`) — submit usage measurements
  - Same `M3terClient`; ingest base URL auto-derived from config API URL
  - Measurements use entity codes (not UUIDs) for submission

**Tavily Web Search:**
- Used by: `backend/app/agents/nodes/use_case_research.py`
- Purpose: Web search to find company pricing/billing info during use case generation
- SDK: `langchain-tavily>=0.1.0` — `TavilySearchResults` tool via LangChain
- Auth: `TAVILY_API_KEY` env var

**LangSmith Tracing:**
- Used by: all LangGraph/LangChain graph invocations (automatic when enabled)
- Purpose: Observability — traces every node execution, LLM call, and tool invocation
- SDK: `langsmith>=0.3.0`
- Auth: `LANGSMITH_API_KEY` env var
- Activation: `LANGSMITH_TRACING=true` env var (purely opt-in; no code changes needed)
- Endpoint: `https://api.smith.langchain.com` (configurable via `LANGSMITH_ENDPOINT`)
- Project: defaults to `"mira"` (configurable via `LANGSMITH_PROJECT`)

## Data Storage

**Databases:**
- Supabase (PostgreSQL 15.8) — primary data store for all application data
  - Connection (application queries): Supabase Python client in `backend/app/db/client.py`
    - Service role client (bypasses RLS): `get_supabase_client()`
    - Anon client (respects RLS): `get_supabase_anon_client()`
  - Connection (LangGraph checkpointer): asyncpg direct pool in `backend/app/db/client.py` (via `DATABASE_URL`)
  - Connection (LangGraph store + checkpointer): separate psycopg pool in `backend/app/agents/checkpointer.py` (min 2, max 15 connections)
  - Connection (RAG vector queries): asyncpg pool via `get_db_pool()`
  - Env vars: `SUPABASE_URL`, `SUPABASE_ANON_KEY`, `SUPABASE_SERVICE_ROLE_KEY`, `SUPABASE_JWT_SECRET`, `DATABASE_URL`
  - Extensions: `pgvector` (cosine similarity on `embeddings` table with HNSW index), `pgcrypto` (UUID generation)
  - Migrations: `backend/migrations/001_extensions.sql` through `013_*.sql`

**Vector Store:**
- pgvector (`vector` extension on Supabase Postgres)
  - Table: `embeddings` — 1536-dimensional vectors, HNSW index (`m=16, ef_construction=64`), cosine ops
  - Two source types: `m3ter_docs` (global, scraped from docs.m3ter.com) and `user_document` (project-scoped)
  - Queried via raw asyncpg in `backend/app/rag/retriever.py`
  - Embeddings generated by OpenAI `text-embedding-3-small`

**File Storage:**
- Local filesystem — uploaded documents stored at `backend/uploads/` (configured via `upload_dir` setting)
- No cloud object storage (S3/GCS) — text extracted in-memory, only chunks/embeddings persisted to DB

**Caching:**
- In-process only — no Redis or external cache
- m3ter OAuth2 tokens: class-level dict on `M3terClient` (4h50m TTL)
- Supabase and asyncpg clients: module-level singletons

## Authentication & Identity

**Auth Provider: Supabase GoTrue**
- Hosted Supabase auth (production) or local Docker `supabase/gotrue:v2.168.0` (dev, port 54321)
- Frontend SSR integration: `@supabase/ssr` in `frontend/src/hooks.server.ts`
  - `createServerClient` on every request; `safeGetSession()` calls `getUser()` first (validates JWT server-side before `getSession()`)
  - Only `sb-` prefixed cookies passed from server to client layouts
- Backend JWT verification: `backend/app/auth/jwt.py`
  - Supports HS256 (shared secret via `SUPABASE_JWT_SECRET`) and ES256 (JWKS from `{SUPABASE_URL}/auth/v1/.well-known/jwks.json`)
  - Custom `AuthError` exception with HTTP status code; registered as FastAPI exception handler in `backend/app/main.py`
- Route protection: `frontend/src/routes/(app)/+layout.server.ts` redirects to `/login` if no session
- WebSocket auth: token passed as query param (`?token=...`), verified via `verify_token()` in `backend/app/api/ws.py`

## Environment Configuration

**Required env vars (Backend — `backend/.env`):**
- `SUPABASE_URL` — Supabase project URL
- `SUPABASE_ANON_KEY` — Supabase anon/public key
- `SUPABASE_SERVICE_ROLE_KEY` — Supabase service role key (bypasses RLS)
- `SUPABASE_JWT_SECRET` — JWT signing secret for HS256 token verification
- `DATABASE_URL` — Direct Postgres connection string (for asyncpg pools)
- `OPENAI_API_KEY` — OpenAI API key (LLM calls + embeddings)
- `ANTHROPIC_API_KEY` — Anthropic API key
- `GOOGLE_API_KEY` — Google GenAI API key
- `ENCRYPTION_KEY` — Fernet key for encrypting m3ter credentials at rest
- `TAVILY_API_KEY` — Tavily web search API key

**Optional env vars (Backend):**
- `LANGSMITH_TRACING` — set to `"true"` to enable LangSmith tracing
- `LANGSMITH_API_KEY` — LangSmith API key
- `LANGSMITH_PROJECT` — LangSmith project name (defaults to `"mira"`)

**Required env vars (Frontend — `frontend/.env`):**
- `PUBLIC_SUPABASE_URL` — Supabase URL (exposed to browser)
- `PUBLIC_SUPABASE_ANON_KEY` — Supabase anon key (exposed to browser)
- `PUBLIC_API_URL` — Backend FastAPI base URL (e.g., `http://localhost:8000`)
- `PUBLIC_WS_URL` — Backend WebSocket base URL (e.g., `ws://localhost:8000`)

**Secrets storage:**
- Backend env vars in `backend/.env` (gitignored)
- m3ter API credentials in database `org_connections` table, encrypted with Fernet before insert

## WebSocket Endpoints

Four real-time WebSocket endpoints served from `backend/app/api/ws.py`:

- `ws://host/ws/workflows/{workflow_id}?token={token}` — bidirectional; LangGraph HITL workflow streaming (start, resume, interrupt delivery)
- `ws://host/ws/push/{use_case_id}?token={token}` — server-push; real-time m3ter entity push progress
- `ws://host/ws/documents/{project_id}?token={token}` — server-push; document processing stage updates
- `ws://host/ws/generate/{project_id}?token={token}` — bidirectional; use case generation with Tavily search and optional clarification
