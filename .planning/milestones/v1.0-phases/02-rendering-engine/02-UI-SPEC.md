# Phase 2 — UI Design Contract

> Visual and interaction contract for the rendering engine phase. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | shadcn-svelte 1.1 (components.json detected) |
| Preset | default style, zinc base color |
| Component library | bits-ui 2.16 (headless primitives) |
| Icon library | lucide-svelte 0.575 |
| Font | Inter (system fallback: -apple-system, BlinkMacSystemFont, sans-serif) |

**Phase 2 dual context:** This phase has two rendering domains:
1. **Application chrome** (page layout, dialog, buttons) -- uses shadcn-svelte + Tailwind as normal
2. **SVG diagram canvas** (the rendered diagram) -- uses pure inline SVG styles with hex values, no Tailwind classes, no foreignObject (REND-03)

All SVG-internal values below are specified as inline style constants, not Tailwind tokens.

## Interaction Contract

### SVG Diagram Rendering (REND-01)

The diagram re-renders reactively when `DiagramContent` changes. The SVG component uses `$derived` to recompute layout positions from the current systems and connections arrays.

- **No drag-and-drop** -- this is a configurator, not a WYSIWYG canvas (per PROJECT.md decision)
- **No click-to-select** on SVG nodes in Phase 2 -- node interaction comes in Phase 3
- **Read-only SVG** -- the canvas is a live preview; modifications come through the configurator UI in Phase 3

### Hub-and-Spoke Auto-Layout (REND-04, D-01/D-02/D-03)

Layout algorithm is a pure function: `layoutDiagram(content: DiagramContent, components: ComponentLibraryItem[]) => LayoutResult`

- Prospect node: fixed at `(HUB_CENTER_X, PROSPECT_Y)`
- m3ter hub: fixed at `(HUB_CENTER_X, HUB_CENTER_Y)`
- System categories: assigned to 5 zones (left, bottom-left, bottom, bottom-right, right) in clockwise order by `display_order`
- Each zone has a center angle and radial distance from hub
- Systems within a zone stack vertically or form a group card
- Canvas does not scroll -- all content fits within the 1200x800 viewBox
- When system count grows, cards shrink proportionally and spacing tightens (reflow, per D-02)

### Custom System Dialog (COMP-02, D-10/D-11)

1. User clicks "Add Custom System" button (icon-only "+" button with `aria-label="Add Custom System"`, visible on the diagram editor page toolbar)
2. Dialog opens with name and domain fields
3. On domain field blur: call `GET /api/logo-proxy?domain={value}` asynchronously
4. Show Skeleton placeholder while loading, then show fetched logo or monogram fallback
5. User clicks "Add System" -- system is added to `DiagramContent.systems[]` with position auto-assigned
6. SVG re-renders reactively with the new node

### Monogram Rendering (Phase 1 format: `monogram:<INITIALS>:<COLOR>`)

When `logo_base64` starts with `monogram:`:
1. Parse format: `monogram:AB:#FF5722`
2. Render as SVG: colored circle (fill = parsed color) with white centered text (parsed initials)
3. Same dimensions as logo (LOGO_SIZE x LOGO_SIZE)

### Connection Line Rendering (D-04/D-05/D-06, CONN-04)

1. Calculate line from source node center to target node center
2. Apply `stroke-dasharray: 6,4` and `stroke-width: 2`
3. Color line by `connection_type` from the color map
4. Place dot marker (`<circle r="4">`) at source end
5. Place arrowhead marker at target end
6. For bidirectional connections: arrowheads at both ends, no dots
7. Calculate midpoint of line
8. Render ConnectionPill at midpoint with label text and background color matching connection type

### States

| State | Visual Treatment |
|-------|-----------------|
| Diagram loading | Skeleton placeholder matching canvas dimensions |
| Diagram loaded (empty -- new diagram) | Canvas shows m3ter hub + prospect node, no connections |
| Diagram loaded (with data) | Full rendering: all nodes positioned by layout algorithm, all connections drawn |
| Logo fetching (in dialog) | Skeleton pulse animation in logo preview area |
| Logo fetched successfully | Shows base64 image in preview area |
| Logo fetch failed | Shows monogram fallback in preview area + muted message |
| Diagram load error | Error state with heading + body + back link |
